# Profile optimization program

## Fixed requirements

- Make the profile visually distinctive without hiding the actual work.
- Serve recruiters and open-source maintainers.
- Exclude the hiring-assessment repository from every showcase layer.
- Keep all claims grounded in public repository evidence.
- Keep the profile readable if images, third-party widgets, or dynamic requests fail.

## Evaluation loop

Each revision is scored from 1–5 on:

1. First-screen visual identity.
2. Recruiter scanability in 30 seconds.
3. Maintainer trust and contribution clarity.
4. Project evidence and click-through usefulness.
5. Mobile and dark/light readability.
6. Maintainability and truthful automation.

The current revision prioritizes identity, visual rhythm, project cards, and open-source positioning while keeping the generated telemetry region isolated.
