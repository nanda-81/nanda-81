# Open-source roadmap

## Stage 1 — vocabulary and local confidence

- Clone a repository and inspect its README, license, contribution guide, issues, and pull requests.
- Create a branch, make one focused change, inspect `git diff`, and write a meaningful commit message.
- Rebase or merge the branch locally and understand what changed.

## Stage 2 — first contribution

- Choose a beginner-friendly issue or documentation improvement.
- Reproduce the problem or verify the missing explanation.
- Make the smallest useful change.
- Add a test or verification note when the change affects behavior.
- Open a pull request with context, implementation summary, and verification steps.

## Stage 3 — review and collaboration

- Read every review comment before replying.
- Ask for clarification when a requirement is ambiguous.
- Push focused follow-up commits and keep the pull request easy to review.
- Thank maintainers and record the engineering lesson learned.

## Stage 4 — sustained contribution

- Contribute documentation, tests, bug fixes, and small features across repositories.
- Learn release notes, changelogs, semantic versioning, licenses, security reporting, and maintainer workflows.
- Use the profile README as an evidence log, not a claim log.
